# Dashnex Application Template

## Overview

This is a **Dashnex application template** — a modular Next.js 16 + Tailwind CSS v4 application deployed as a Cloudflare Worker via OpenNext. The application is powered by `@dashnex/core`, which discovers and loads Dashnex modules — each providing React pages, API routes, menu items, widgets, event listeners, CLI commands, and database models.

All `@dashnex/*` modules are published to the private npm registry at **npm.dashnex.com** and installed as regular npm dependencies. Local development modules can be linked via `link:packages/<name>` in `package.json`.

## Quick Start

```bash
# Login to Dashnex
dashnex login

# Create a new application from the template
dashnex app create

# Install dependencies (uses npm.dashnex.com registry)
dashnex install

# Build the module registry (auto-runs via postinstall)
dashnex build

# Create/Update local database
dashnex db migrate

# Start the dev server
dashnex dev

# Push changes
dashnex push

# Run deployment
dashnex deploy
```

## Architecture

### Key Generated Files (src/dashnex/ — gitignored, do NOT edit)

| File | Purpose |
|------|---------|
| `client.ts` | Imports all modules' `./client` entry, builds client-side `ModuleRegistry`, instantiates `DashnexCore` |
| `server.ts` | Imports all modules' default (server) entry, builds server-side `ModuleRegistry`, instantiates `DashnexCore` |
| `worker.ts` | Cloudflare Worker entry — routes requests through DashnexCore or falls back to Next.js |
| `dashnex.css` | Tailwind `@source` directives for all module `dist/` directories + CSS imports |

### Import Patterns

Modules expose two entry points via `package.json` exports:

```typescript
// Server-side (API routes, services, models)
import { getDatabase, getBinding, getEvents, getMailer } from '@dashnex/core'
import { getAuth, getUser, getUsers } from '@dashnex/auth'
import { getOption } from '@dashnex/system'

// Client-side (React components, hooks)
import { Page, getMenu, getEvents } from '@dashnex/core/client'
import { AuthProvider, useAuth, RoleGuard } from '@dashnex/auth/client'
import { Upload, Editor, ComingSoon, AuthPage } from '@dashnex/system/client'
import { Button, Input, Dialog, SidebarLayout, Table } from '@dashnex/ui/client'
import { HomeIcon, UserIcon } from '@dashnex/ui/client'
```

### 4. dashnex.json — Module Manifest

This file is required for module discovery. It can be empty `{}` or declare:

```json
{
  "bindings": {
    "MY_DB": { "type": "DB", "description": "Module database" },
    "MY_STORAGE": { "type": "STORAGE", "description": "File storage" },
    "MY_QUEUE": { "type": "QUEUE" },
  }
}
```

Binding types: `AGENT`, `OBJECT`, `WORKER`, `AI`, `RAG`, `DB`, `QUEUE`, `STORAGE`.
Existing modules already provide some bindings which can be read by

```bash
dashnex bindings
```

### 5. Defining Menu Items

Dynamic menu items via the `system.menu_get` event listener:

```typescript
export async function onMenuGet(context: MenuGetContext) {
  if (context.user && hasRole(context.user, 'admin')) {
    context.items.push({
      name: 'Dynamic Item',
      position: 'sidebar',
      path: '/admin/my-module/dynamic',
      roles: ['admin'],
      icon: UserIcon,
    })
  }
}
```

### 6. Database Models

Use Drizzle ORM with SQLite (Cloudflare D1):

```typescript
import { sqliteTable, text, integer } from 'drizzle-orm/sqlite-core'
import { createId } from '@paralleldrive/cuid2'
import { date } from '@dashnex/core'

export const myItems = sqliteTable('my_items', {
  id: text('id').primaryKey().$defaultFn(() => createId()),
  name: text('name').notNull(),
  createdAt: date('created_at').$defaultFn(() => new Date()),
  updatedAt: date('updated_at').$defaultFn(() => new Date()),
})

export type MyItem = typeof myItems.$inferSelect
export type NewMyItem = typeof myItems.$inferInsert
```

Access the database in route handlers/services:

```typescript
import { getDatabase } from '@dashnex/core'
import { myItems } from '../models/my-item.model.js'

const db = getDatabase()
const items = await db.select().from(myItems).all()
```

Place SQL migrations in `migrations/` (e.g., `001_create_my_items.sql`). Run with `dashnex db migrate`.


### 7. Event System

Register listeners in the `ModuleConfig.listeners` array:

```typescript
{ event: 'core.request', handler: myHandler, priority: 10 }
```

Higher priority runs first. Available core events:

**Server events:** `core.request`, `core.route`, `core.params`, `core.error`, `core.response`, `core.before_mailer_send`, `core.before_send_transactional`, `core.before_email_render`, `core.email_layouts_get`, `core.menu_get`, `core.widgets_get`

**Client events:** `core.before_page_load`, `core.after_page_load`

**System events:** `system.menu_get`, `system.admin_settings_get`, `system.layouts_get`, `system.user_account_get`, `system.before_page_load`, `system.scripts_render`, `system.metadata_load`

**Auth events:** `auth.login`, `auth.user_format`, `auth.before_user_register`, `auth.after_user_register`, `auth.confirmed`, `auth.oauth_login`

Emit custom events:

```typescript
import { getEvents } from '@dashnex/core'
const events = getEvents()
await events.emit('my-module.item_created', { item })
```

## Coding Conventions

- **Tailwind CSS only** for styling — no custom CSS, use `@dashnex/ui` components
- **No semicolons**, single quotes, 120 char line width (see `prettier.config.mjs`)
- **`@dashnex/ui/client`** for all UI components — check existing pages for HTML structure patterns
- **Zod** for request/input validation in services
- **`@paralleldrive/cuid2`** for generating IDs
- **Drizzle ORM** for database models (SQLite / D1)
- **Pass model objects** to functions, not just IDs (e.g., pass `broadcast` object, not `broadcastId`)
- **API response format**: `{ success: true|false, data?: object, error?: string }`
- **Route handlers** accept `(request: Request, params?: any)` — `params` contains path placeholders
- **Role hierarchy**: `owner` > `admin` > `user` > `guest`
- **`"use client"`** directive required on all React page components rendered via the module page system
- **Event handlers** named with `on` prefix (e.g., `onCoreRequest`, `onMenuGet`)
- **Vite** for module builds with `rollup-plugin-preserve-directives` (preserves `"use client"`)
- Module builds output ES modules with `preserveModules: true` for tree-shaking
