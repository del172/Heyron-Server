## Key Discovery: Firecrawl Session Persistence

**Problem:** Firecrawl wasn't maintaining login sessions - kept getting logged out.

**Solution:** Use SAME scrapeId for all interact calls:
1. Scrape login page ONCE → get scrapeId
2. Use that scrapeId for ALL subsequent interact calls
3. NEVER make new scrape requests after login

```bash
# Step 1: Get scrapeId
SCRAPE_ID=$(curl -s ... | jq -r '.data.metadata.scrapeId')

# Step 2: Login with SAME scrapeId
curl -s -X POST "https://api.firecrawl.dev/v2/scrape/$SCRAPE_ID/interact" \
  -d '{"prompt": "Fill email ... click Next"}'

# Step 3: Continue with SAME scrapeId
curl -s -X POST "https://api.firecrawl.dev/v2/scrape/$SCRAPE_ID/interact" \
  -d '{"prompt": "Fill password ... click Sign in"}'
```

## Dashnex Custom Domain Required

**Finding:** The .dashnexcloud.com subdomain has LIMITED features:
- ❌ No Website Builder
- ❌ No Funnels/Pages editing
- ❌ No Offers configuration

**Solution:** Need to connect a custom domain to unlock full features.

### How to Add Custom Domain (from Dashnex docs):
1. Buy domain (GoDaddy, Namecheap, Cloudflare)
2. Go to Dashnex Dashboard → **Domains**
3. Click **Integrate with CloudFlare** → enter CloudFlare credentials
4. Click **Add domain** → enter your domain
5. Update nameservers at your registrar
6. Wait 24-48hrs for validation
7. Click **Assign Property** → select "Brain Training for Dogs"
8. Go to **Website Settings** → **Custom Domain** → select home page
